# Пакет lidars_localization 

На данный момент этот пакет предназначен для запуска соединения с двумя лидарами и поднятия картографирования для мобильного робота.

## /config

Конфиги для RViz2, slam_toolbox, мерджера сканов и самих лидаров

### /config/lidars/lidars_params.yaml

`lidars_params.yaml` - описание дефолтных значений для лидаров (неймспейсы, фреймы и углы сканирования) и мерджера сканов (фреймы, топики и прочая детальная настройка).

### /config/rviz2/rviz2_config.rviz

`rviz2_config.rviz` - парметры запуска для RViz2. Используется только для дебага.

### <a id="slam_params.yaml">/config/slam/slam_params.yaml</a>

`slam_params.yaml` - дефолтные настройки для узла решения задачи SLAM.

## /launch

### /launch/lidars_manager.launch.py

`lidars_manager.launch.py` - скрипт для открытия взаимодействия с обоими лидарами через ROS2.

В данном launch-файле вызывается скрипт `get_lidars_ports.py` для определения портов и создаются ноды для контактирования с лидарами (запуск скрипта `urg_node2.launch.py` из пакета `urg_node2`).

Скрипт `lidars_manager.launch.py` вызывается только при работе с реальными устройствами (подробнее в описании скрипта [`main.launch.py`](#main.launch.py))

## lidars_localization

### /lidars_localization/laser_scan_merger.py

`laser_scam_merger.py` - скрипт для узла склеивания сканов двух лидаров в один общий скан.

Нода получает сканы от двух лидаров, трансформирует каждый скан в точки в общем фрейме (`target_frame`) координат (используя `tf2`), затем фильтрует дублирующиеся точки (оставляя ближайшие к источнику) и собирает все точки обратно в единый LaserScan. Результирующий скан покрывает объединенную область видимости обоих лидаров с учетом их взаимного расположения в пространстве.

## lidars_localization

### /lidars_localization/get_lidars_ports.py

`get_lidars_ports.py` - скрипт для определения `/dev/ttyACM*` портов, к которым подключены лидары. 
Определение портов происходит путем прослушивания всех существующих `/dev/ttyACM*` и определения, на каких из них есть лидары HOKUYO URG.
Орпеделние портов лидаров осуществляется с помощью серийных номеров обоих лидаров.

Серийный номер переднего лидара: ***H1766497***   
Серийный номер заднего лидара: ***H1766499***

Возвращает строки с названиями портов, к корым подключены лидары. Формат: `['порт_переднего_лидара', 'порт_заднего_лидара']`.   

Таким образом, лидары менять местами на самом роботе *НЕЛЬЗЯ (ЗАПРЕЩЕНО)*. 
Тот, который должен быть спереди (***H1766497***) - должен находиться спереди.
Тот, который должен быть сзади (***H1766499***) - должен находиться сзади.

Данный скрипт предназначен *ИСКЛЮЧИТЕЛЬНО* для того, чтобы оба лидара можно было подключать к любым физическим USB портам на неттопе и не бояться, что лидары перепутаются.
