# Инструкция по использованию репозитория

## Структура репозитория
- **docker/** — папка со скриптами для работы с Docker-контейнером:
  - **docker/build_docker** — скрипт для построения Docker-образа.
  - **docker/start_docker** — скрипт для запуска Docker-контейнера.
  - **docker/attach_docker** — скрипт для подключения к командной строке внутри Docker-контейнера.
  - **docker/stop_docker** — скрипт для остановки Docker-контейнера.
  - **docker/save_image** — скрипт для сохранения текущего Docker-образа в файл.
  - **docker/load_image** — скрипт для загрузки Docker-образа из файла.
  - **docker/.bashrc** — скрипт, выполняющийся при подключении к командной строке контейнера (инициализация ROS и проекта).
  - **docker/dockerfile** — описание Docker-образа (файл Dockerfile).
- **ws/** — рабочая папка с ROS-проектами. В контейнере она монтируется по адресу `/home/$USER/local/workspace/ws`.

Сборка проекта внутри Docker является опциональной. Проект можно собирать в нативной системе, установив все зависимости вручную. Docker-образ включает предустановленные зависимости, что упрощает процесс работы. В контейнере используется Ubuntu и ROS Humble.

## Использование Docker

### 0. Установка Docker
1. Установите Docker по [официальной инструкции](https://docs.docker.com/engine/installation/linux/docker-ce/ubuntu/#os-requirements).
2. Настройте Docker для работы без `sudo`:
```bash
   $ sudo groupadd docker
   $ sudo usermod -aG docker $USER
   $ systemctl is-enabled docker
```
После этого перезайдите в систему, чтобы изменения вступили в силу.

1. Клонирование репозитория
Склонируйте репозиторий:
```bash
   $ git clone https://github.com/DedBanzay/docker_template.git <папка для размещения проекта>
```
Все дальнейшие команды выполняйте из директории с проектом под обычным пользователем.

2. Сборка Docker-образа
Сборка Docker-образа выполняется один раз или при изменении файла Dockerfile:
``` bash
   $ docker/build_docker
```
Процесс может занять некоторое время, так как потребуется скачать базовый образ и установить зависимости.

3. Запуск Docker-контейнера

Запустите контейнер:
``` bash	
   $ docker/start_docker
```
Контейнер создаёт пользователя с именем текущего пользователя системы ($USER).
Текущая директория проекта монтируется в контейнер по адресу /home/<имя пользователя>/local/workspace.
Домашняя папка текущего пользователя монтируется в контейнер по адресу /host.

4. Подключение к контейнеру

Для входа в контейнер используйте команду:
``` bash
   $ docker/attach_docker
```
При выполнении этой команды терминал подключается к командной строке внутри контейнера. Вы окажетесь в директории ROS workspace (/home/<имя пользователя>/local/workspace/ws).
Для первого запуска потребуется собрать проект и проинициализировать workspace:
``` bash
   $ colcon build
   $ source install/setup.bash
```
При последующих запусках workspace будет инициализироваться автоматически.

5. Сохранение Docker-образа

Чтобы сохранить текущий Docker-образ в файл, выполните:
``` bash
   $ docker/save_image <путь к файлу>
```
Это создаст файл с образом, который можно перенести на другой компьютер.

6. Загрузка Docker-образа

Для загрузки Docker-образа из файла используйте:
``` bash
   $ docker/load_image <путь к файлу>
``` 
Это восстановит ранее сохранённый Docker-образ.

7. Запуск контейнера из загруженного образа

После загрузки Docker-образа можно запустить контейнер с помощью команды:
``` bash
   $ docker/start_docker
```
Контейнер будет запущен из загруженного образа с сохранением всех настроек.

8. Запуск проектов

Запуск ROS-проектов осуществляется с помощью команд:
``` bash
   $ source install/setup.bash
   $ ros2 run pkg_name launchfile_name
```

Также для запуска проекта можно использовать скрипт start.sh. При этом откроется окно симулятора Gazebo и инструмент rqt для управления проектом (например, задания скорости движения робота).

9. Завершение работы с контейнером

Чтобы выйти из контейнера, выполните:
``` bash
   $ exit
```
Или просто закройте терминал.

10. Просмотр запущенных контейнеров

Для просмотра всех запущенных контейнеров используйте команду:
``` bash
   $ docker ps
```

11. Остановка контейнера

Для остановки запущенного контейнера выполните:
``` bash
   $ docker/stop_docker
```

12. Важно

Все изменения, сделанные внутри контейнера (например, установка новых пакетов), кроме изменений в папке проекта (/home/<имя пользователя>/local/workspace), будут утеряны после его остановки.
При выключении или перезагрузке компьютера все контейнеры также будут остановлены.
